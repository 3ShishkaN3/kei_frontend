# Kei Frontend — Фронтенд платформы Keisenpai

Этот проект представляет собой SPA (Single Page Application), разработанное на **Svelte 4**. Фронтенд отвечает за визуализацию учебного процесса, интерактивные карточки слов, прохождение уроков и отображение прогресса пользователя.

## Технологический стек

*   **Фреймворк**: Svelte 4
*   **Сборщик**: Rollup
*   **Стили и UI**: Material UI (MUI) для Svelte, FontAwesome
*   **Анимации**: GSAP (GreenSock Animation Platform) + ScrollTrigger для плавных переходов и интерактивности
*   **Роутинг**: `svelte-routing`
*   **Состояние**: Svelte Stores
*   **Графики**: Chart.js
*   **Интеграции**: WebPush API для уведомлений

## Разработка

### Локальный запуск (Рекомендуется)

Во время разработки **крайне рекомендуется запускать фронтенд локально**, а не через Docker. Контейнер может некорректно обрабатывать изменения в файлах (HMR), что замедляет работу.

1.  Убедитесь, что у вас установлен Node.js.
2.  Перейдите в папку проекта:
    ```bash
    cd kei_frontend
    ```
3.  Установите зависимости:
    ```bash
    npm install
    ```
4.  Запустите сервер разработки:
    ```bash
    npm run dev
    ```
5.  Откройте [http://localhost:5000](http://localhost:5000).

*Примечание: Бэкенд должен быть запущен параллельно через `make up` в корне проекта.*

### Сборка

Для создания оптимизированной сборки для продакшена:
```bash
npm run build
```
Результат будет находиться в папке `public/build`.

## Структура проекта

*   **`src/api/`**: Логика взаимодействия с бэкендом (Axios/Fetch запросы).
*   **`src/components/`**: Переиспользуемые UI компоненты (кнопки, карточки, модалки).
    *   `admin/`: Компоненты для панели администратора.
    *   `utils/`: Вспомогательные компоненты (уведомления, лоадеры).
*   **`src/routes/`**: Основные страницы приложения (Home, Profile, Lesson, Practice и др.).
*   **`src/stores/`**: Состояние приложения (данные пользователя, текущий курс, настройки).
*   **`src/lib/`**: Вспомогательные библиотеки и утилиты (например, работа с WebPush).
*   **`public/`**: Статические файлы, логотипы, глобальные CSS стили.

## Особенности реализации

### Анимации
Для создания сложных анимаций используется библиотека **GSAP**. Анимации прописаны в компонентах для появления элементов при скролле или смене состояний.

### State Management
В проекте активно используются **Svelte Stores** (в папке `src/stores`). Например, `user.js` хранит состояние авторизации, что позволяет легко проверять доступ к роутам в `App.svelte`.

### Уведомления
Реализована поддержка Push-уведомлений. Логика подписки находится в `src/lib/webpush.js`, а сами уведомления отображаются через `NotificationsContainer.svelte`.
